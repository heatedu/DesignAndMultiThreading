## ğŸ¯ SOLID Principles Applied

### Single Responsibility Principle (SRP)
```
âœ… Elevator: Manages single elevator state
âœ… ElevatorController: Coordinates multiple elevators
âœ… SchedulingStrategy: Determines next stop only
âœ… ElevatorDisplay: Displays information only
```

### Open/Closed Principle (OCP)
```
âœ… SchedulingStrategy interface
   - Open for extension (new algorithms)
   - Closed for modification (interface stable)
   
âœ… ElevatorObserver interface
   - Can add new observers without changing Elevator
```

### Liskov Substitution Principle (LSP)
```
âœ… Any SchedulingStrategy can replace another
   - FCFS, SCAN, LOOK are interchangeable
   
âœ… Any ElevatorObserver can replace another
   - Display, Logger, Monitor are interchangeable
```

### Interface Segregation Principle (ISP)
```
âœ… Small, focused interfaces
   - ElevatorObserver: Only 2 methods
   - ElevatorCommand: Only execute()
   - SchedulingStrategy: Only getNextStop()
```

### Dependency Inversion Principle (DIP)
```
âœ… High-level modules depend on abstractions
   - Controller depends on SchedulingStrategy (interface)
   - Elevator depends on ElevatorObserver (interface)
   - Not on concrete implementations
```

## ğŸ” Key Design Decisions

### 1. Queue vs Priority Queue
**Decision**: Use `std::queue` for simplicity  
**Rationale**: Easier to understand in interview, can extend to priority queue  
**Trade-off**: Less flexible for priority handling

### 2. Raw Pointers vs Smart Pointers
**Decision**: Mix of both  
- Raw pointers for observers (non-owning)
- Smart pointers for strategies (owning)  
**Rationale**: Clear ownership semantics

### 3. Internal vs External Requests
**Decision**: Single `ElevatorRequest` class with flag  
**Rationale**: Avoids duplication, easy to extend  
**Alternative**: Separate classes (more OOP but more complex)

### 4. Synchronous vs Asynchronous
**Decision**: Synchronous simulation  
**Rationale**: Simpler for interview, focus on logic not threading  
**Extension**: Can add threading for real system
